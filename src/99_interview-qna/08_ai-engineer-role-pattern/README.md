# AI engineer — interview track

Executable pack for **individual-contributor** AI engineering: you **ship and operate** an LLM-backed feature (service boundary, APIs, retrieval, prompts, tests, observability)—not the whole company platform or enterprise portfolio.

## How this differs from nearby folders

| Track | Emphasis |
|-------|----------|
| **`08_ai-engineer-role-pattern/`** (here) | **One feature / service**: contracts, latency, errors, basic eval, logging, cost awareness—typical **Senior AI engineer** or **strong mid-level** shipping loop. |
| **`ai-systems-architect/`** | **Systems architect** breadth: `complete.md` + serving/observability at **architecture** depth. |
| **`04_agentic-ai-lead-competency-profile/`** | **Lead-level** multi-agent + RAG orchestration, more moving parts and team-scale narrative. |
| **`07_gen-ai-lead-role-pattern/`** | **Platform**: registry, lifecycle, org-wide governance. |
| **`career-orientation/01_production-ai-engineer-skill-map.md`** | Short **orientation**; pair with this folder for drills. |

Shared spine: `../interview-execution/`.
