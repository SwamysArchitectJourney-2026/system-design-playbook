# Interview prep — master index

This folder is the **published interview serving layer**: role tracks, shared execution spine, and orientation. **High churn and mission-critical** for prep—not an archive.

---

## Why the folder is named `99_interview-qna`

The **`99_`** prefix sorts **interview prep** **after** `src/01_foundations/` through `07_case-studies/` so the **curriculum scaffolds** keep stable numbering. That choice can **look** like “misc” or “legacy”; **functionally**, this directory is **primary** for interview readiness. If you only open one subtree under `src/` before a loop, it is reasonable for this to be the one.

---

## Daily workflow (pick → spine → drill)

1. **Choose a role** — open that track’s `README.md` from the [role tracks](#role-tracks) table (or [specialty packs](#specialty-and-long-form-packs) below).
2. **Run the shared execution spine** — [golden template](interview-execution/01_golden-answer-template.md) → [grill layer](interview-execution/02_follow-up-grill-layer.md); optional [cross-domain prompts](interview-execution/03_cross-domain-prompt-bank.md). *Payment-specific spine:* [payment-system-interview/interview-execution/](payment-system-interview/interview-execution/README.md).
3. **Drill the role pack** — that folder’s `01_templates/`, `02_core_questions/<slug>/`, and any whiteboard or troubleshooting files linked from the track README.

**Governance:** [01_interview-prep-conventions.md](01_interview-prep-conventions.md) — track layout contract, **duplication policy**, link hygiene.
