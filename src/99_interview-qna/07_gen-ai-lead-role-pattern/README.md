Executable pack for **Generative AI lead**–style loops: **model lifecycle**, **LLMOps**, **evaluation gates**, **responsible AI / governance**, and how you run **inference + training** platforms—not only a single RAG app.

## How this differs from nearby folders

| Track | Emphasis |
|-------|----------|
| **`04_agentic-ai-lead-competency-profile/`** | Production **RAG + agent orchestration** (retrieval, tools, multi-step workflows). |
| **`06_enterprise-principal-genai-and-ea-themes/`** | **Portfolio / modernization / EA** narratives across programs. |
| **`07_gen-ai-lead-role-pattern/`** (here) | **Platform and lifecycle**: registry, training/fine-tuning jobs, deployment, monitoring, safety, cost—what a **Gen AI lead** often owns end-to-end. |
| **`08_ai-engineer-role-pattern/`** | **IC feature** shipping—API, retrieval slice, eval hooks—not org-wide LLMOps platform. |
| **`career-orientation/02_gen-ai-lead-competency-compass.md`** | Short **orientation** map; use with this folder for drills. |

Shared spine: `../interview-execution/`.

For **smaller IC scope** (one shipped feature, not a platform), use `../08_ai-engineer-role-pattern/README.md`.
