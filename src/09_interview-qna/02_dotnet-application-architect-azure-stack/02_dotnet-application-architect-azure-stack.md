# .NET application architect on Azure: competency pattern

*Aggregated themes from senior application-architect postings that pair **.NET delivery** with **Azure operations**.*

## Role shape

- **Hands-on architect** embedded with Scrum teams: backlog, design spikes, **unblocking production issues**.
- **Research and POCs** for modernization (cloud patterns, new Azure services, front-end or integration changes).
- **Guardrails** — enterprise standards, reusable patterns, code and design reviews.

## Hard skills that repeat

- **C# / .NET** at expert level; common pairing with **Angular** or similar for full-stack discussions.
- **Azure services** — App Services or Functions, **API Management**, Azure SQL, **Cosmos DB**, Blob storage, **Service Bus**, Event Grid.
- **Architecture vocabulary** — Microservices, event-driven and serverless, REST design, classic patterns applied at scale.
- **DevOps** — Azure DevOps pipelines, IaC exposure, Git branching and release strategy.
- **Security** — RBAC, secure coding, **Entra ID (Azure AD)**, OAuth2/OIDC flows.

## Emerging expectations

- **AI-assisted development** — productive use of Copilot-class tools without bypassing review and security.
- **Python** sometimes listed as a plus for data or automation side tasks.

## Domain overlays

Financial or regulated domains (for example mortgage tech) add **auditability, change control, and compliance** questions—prepare examples with traceability and segregation of duties.

## Interview preparation map

| Theme | Be ready to explain |
|-------|---------------------|
| Integration | API Management policies, throttling, versioning |
| Data | When SQL vs Cosmos; consistency and partition keys |
| Events | Service Bus vs Event Grid; ordering and idempotency |
| Identity | OIDC flows, B2B vs workforce, managed identities |
